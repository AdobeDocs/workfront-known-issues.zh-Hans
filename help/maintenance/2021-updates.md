# 2021年 [!DNL Workfront] 维护更新

2021年进行了以下维护更新：

## 2021年12月更新

+++**2021年12月23日的维护更新**

**新任务默认为不正确的任务约束**

_任务_

当用户使用“[!UICONTROL 新任务]“ ”按钮， [!UICONTROL 新任务默认开始] 日期选项设置为“[!UICONTROL 今天]，则创建任务的任务约束设置为“[!UICONTROL 尽快]&quot;而不是&quot;[!UICONTROL 开始时间不早于].&quot; 在使用项目模板时，也可能会发生这种情况。

**在中打开计划 [!UICONTROL 群组] 区域导致空白页面**

_设置_

当用户在 [!UICONTROL 设置] 区域，并尝试打开、编辑或复制计划，计划不会打开，用户会看到空白页面。

**重新排序左侧导航时，不显示更改**

_导航_

当用户尝试通过拖动项目来重新排序左侧导航面板时，当用户放置项目时，该项目会显示在其先前的位置。 如果用户刷新页面，左侧面板将显示新顺序。

**用于提交请求文档的链接将导致空白页面。**

_文档_

当用户收到提交文档的请求，并单击指向请求文档的对象的链接时，该链接将导致空白页面。 在电子邮件或应用程序内通知中单击链接时，可能会发生这种情况。

**[!UICONTROL 工作负载平衡器] 显示已分配的0小时**

_[!UICONTROL 工作负载均衡器]_

当用户查看 [!UICONTROL 工作负载平衡器] 并具有“[!UICONTROL 显示预计日期]“ ”设置启用后，将来的任何日期都会显示已分配0小时。

**校样间歇性地从文件夹中消失**

_[!DNL Workfront Proof]_

登录用户 [!DNL Workfront Proof] 查看文件夹时，文件夹显示为空。 如果用户稍后进行检查，则会显示校样。

+++

+++**2021年12月16日的维护更新**

**单击通知列表中的公告将导致空白页面**

_通知_

当用户从 [!UICONTROL 通知] 图标，然后单击公告，将它们转到空白页面，且不会显示公告。

**摘要面板显示“[!UICONTROL 无选择]选择任务时**

_任务_

当用户在 [!UICONTROL 文档] 项目区域，然后转到任务列表，选择任务，并尝试打开任务摘要，此时不会显示任务摘要，用户会看到以下消息：

[!UICONTROL 无选择项. 选择列表中的一个文档以查看详细信息.]

即使用户在任务列表中，该消息仍会提及文档。

**[!UICONTROL 未分配的工作] 未加载**

_[!UICONTROL 工作负载均衡器]_

当 [!UICONTROL 工作负载平衡器] 使用创建过滤器 [!UICONTROL 分配：角色ID] 字段， [!UICONTROL 未分配的工作] 区域。

**使用“[!UICONTROL 自定义和附加]&quot;选项清除自定义字段值**

_项目_

如果用户使用“[!UICONTROL 自定义和附加]“选项，并且项目已附加自定义表单，则自定义字段值不会结转，必须手动重新输入。 即使模板包含相同的自定义表单，也会发生这种情况。

+++

+++**2021年12月10日的维护更新（热修复程序）**

**[!UICONTROL 哇] 将模板附加到现有项目时出错**

_项目_

当用户尝试将模板附加到现有项目时，该模板不会附加，用户会看到以下错误：

&quot;[!UICONTROL 哇！ 出现了一些问题。请联系 [!DNL Workfront] 这样我们就能弄明白出了什么问题，并解决它。]&quot;

+++**2021年12月9日的维护更新**


**折叠的左侧导航面板在页面加载时展开**

_导航_

当用户将其左侧导航设置为折叠，然后刷新页面时，重新加载的页面上会展开左侧导航。 如果用户在新选项卡中打开页面，则左侧导航也会展开。

**[!UICONTROL 工作负载平衡器] 显示已分配的0小时**

_[!UICONTROL 工作负载均衡器]_

当用户查看 [!UICONTROL 工作负载平衡器] 并具有“[!UICONTROL 显示预计日期]“ ”设置启用后，将来的任何日期都会显示已分配0小时。

+++

+++**2021年12月8日的维护更新**

**进行更新时的批准重置**

_审批_

如果用户使用对象标题对批准做出决策，然后立即对对象进行更新，则批准图标将重新出现在标题中，并且不会保存批准决策。

**[!UICONTROL 无法在内联编辑报表中的分配]**

_报告_

当用户尝试在报表中内联编辑分配时，字段值不会更改，并且编辑也不会保存。

+++**2021年12月2日的维护更新**

**手动键入URL时添加额外的斜杠**

_请求_

当用户填写请求并手动开始键入URL时，会在URL开头附近的某个点添加额外的斜杠。 用户无法删除斜杠。

**无法从左侧导航面板中删除自定义部分**

_导航_

当用户尝试通过单击左侧导航面板旁边的X来从左侧导航面板中删除自定义部分时，不会删除该部分。

**未分配的工作未加载**

_[!UICONTROL 工作负载均衡器]_

当 [!UICONTROL 工作负载平衡器] 使用创建过滤器 [!UICONTROL 分配：角色ID] 字段， [!UICONTROL 未分配的工作] 区域。

**某些浏览器中未加载页面**

_[!DNL Workfront]_

当用户在 [!DNL Workfront]，则页面不会加载，用户会看到以下错误消息：

&quot;[!UICONTROL 出现错误，我们正在努力解决此问题。要继续工作，请尝试刷新此浏览器页面。]&quot;

这已在

* [!DNL Firefox]
* [!DNL Microsoft Edge]

此错误会随机发生，并可能影响 [!DNL Workfront].

## 2021年11月更新

+++**2021年11月18日的维护更新**

**[!DNL Workfront]表示 [!DNL Jira] &quot;[!UICONTROL 客户端ID或客户端密钥无效]登录时出错**

_Workfront集成_

用户已从 [!DNL Jira] 用于Workfront集成。 当用户尝试登录 [!DNL Workfront for Jira] 集成时，用户无法登录，并且会看到以下错误：

&quot;[!UICONTROL 客户端ID或客户端密钥无效]&quot;

**选择新的队列主题后，附加到请求的自定义表单不会更新**

_请求_

当用户创建请求并选择自动将自定义表单附加到请求的队列主题，然后选择其他队列主题时，第一个队列主题中的自定义表单仍会附加到请求。

**图标显示不正确**

_[!DNL Workfront]_

图标图像显示不正确。 这在各地的许多领域都有报道 [!UICONTROL Workfront].

**选择“其他大小”选项时，任务不会导出到PDF。**

_任务_

如果用户尝试将任务列表导出到PDF，并选择“[!UICONTROL 其他大小]“ ”选项，他们可以选择大小并单击 [!UICONTROL 导出]，但列表不会导出。 没有错误消息，也没有其他指示表示导出失败。

**电子邮件通知中未显示图像指示器**

_通知_

当用户向更新添加图像，并向更新的收件人发送电子邮件通知时，电子邮件不会包含更新中存在图像的指示符。

**某些浏览器中未加载页面**

_[!DNL Workfront]_

当用户在 [!DNL Workfront]，则页面不会加载，用户会看到以下错误消息：

&quot;[!UICONTROL 出现错误，我们正在努力解决此问题。要继续工作，请尝试刷新此浏览器页面。]&quot;

这已在

* [!DNL Firefox]
* [!DNL Microsoft Edge]

此错误会随机发生，可能会影响Workfront的任何区域。

+++

+++**2021年11月11日的维护更新**

**文档集成存在问题，文档上传弹出窗口中的空白页面[!DNL Google Drive*]*

_文档_

当用户尝试将新文档添加到 [!DNL Workfront] 从 [!DNL Google Drive] 使用 [!UICONTROL 新增] >[!UICONTROL 从 [!DNL Google Drive]]，则上传弹出屏幕仍为空。

**在工作负载平衡器中不能使用多个过滤器**

_[!UICONTROL 工作负载均衡器]_

当用户尝试在 [!UICONTROL 工作负载平衡器]，则他们会看到以下问题：

* 如果用户选择两个过滤器，则仅应用底部过滤器。
* 如果用户选择两个以上的过滤器，则不会显示任何结果。

**&quot;[!UICONTROL 项目文件夹]“项目文档区域中缺少文档标题”**

_项目_

当用户在项目中并查看项目文档时，标题为“[!UICONTROL 项目文件夹]“ ”在左侧导航中缺失。 下拉箭头仍然在此处，用户可以选择文件夹。

**看板板上的列太宽，无法调整**

_Agile_

当用户查看具有多列的看板板时，这些列太宽，用户必须滚动才能查看或将卡片移动到最右侧的列。 列宽是不可调的，因此用户无法使列变小，以便它们可以同时在屏幕上显示。

**改进了用于创建新团队的界面**

_团队_

现在，通过新的视觉提示，可以更直观地创建团队。 当您选择 [!UICONTROL 切换团队] 图标， [!UICONTROL 新建团队] 链接具有指示“[!UICONTROL 新建]，并且该链接与列表的其余部分分开，因此看起来不像团队名称。 此界面与敏捷团队和非敏捷团队的界面相同。

+++

+++**2021年11月4日的维护更新**

**新任务默认为不正确的任务约束**

_任务_

当用户使用“[!UICONTROL 新任务]“ ”按钮，并且“新建任务默认开始日期”选项设置为“ ”[!UICONTROL 今天]，则创建任务的任务约束设置为“[!UICONTROL 尽快]&quot;而不是&quot;[!UICONTROL 开始时间不早于].&quot;

**字段不显示在敏捷文章卡片上**

_敏捷_

当用户查看Agile情节提要时，卡片仅显示 [!UICONTROL 描述] 和 [!UICONTROL 状态] 字段。 任何其他字段（包括任何自定义字段）均不显示。

**信息卡在移至新列之前会返回到原始列**

_敏捷_

当用户将信息卡拖入序列图像板上的新列时，用户可以看到正在拖动的信息卡。 但是，当用户将信息卡放入新列时，信息卡会在原始列中短暂显示，然后才会在新列中显示。

**值不适用于过滤器中的自定义字段**

_[!UICONTROL 工作负载均衡器]_

当用户尝试使用自定义字段创建过滤器时，该自定义字段的值不会显示，且无法输入到过滤器中。

**页面未在中加载 [!DNL Firefox] 浏览器**

_[!DNL Workfront]_

当用户在 [!DNL Workfront] 使用 [!DNL Firefox] 浏览器，页面未加载，用户看到以下错误消息：

&quot;[!UICONTROL 出现错误，我们正在努力解决此问题。要继续工作，请尝试刷新此浏览器页面。]&quot;

此错误会随机发生，并可能影响 [!DNL Workfront].

**从模板创建项目时出现与日期相关的错误。**

_模板_

如果用户从模板创建项目，并将计划模式设置为 [!UICONTROL 开始日期]，然后选择任务约束，当用户尝试创建项目时，不会创建项目，并且用户会看到基于任务约束的错误消息。

**[!UICONTROL 导出甘特图] 对话无响应**

_甘特图_

如果用户位于 [!DNL Workfront] 体验尝试导出 [!UICONTROL 甘特图] 并选择“[!UICONTROL 我看到的]”选项， [!UICONTROL 甘特图] 无法导出，且对话框无响应。 用户无法关闭或单击对话框外的。

**图标显示不正确**

_[!DNL Workfront]_

图标图像显示不正确。 已报告这种情况，包括但不限于：

* 共享对象时的作业角色或组图像
* 日历报表中的左图标和右图标
* 对报表列中的图标进行排序

**将复选框添加到 [!UICONTROL 已提交] 部分 [!UICONTROL 请求] 面积**

_请求_

我们已在 [!UICONTROL 已提交的列表] 的 [!UICONTROL 请求] 的上界。 这样，您就可以更轻松地选择请求和查看其他信息。

**工作负载平衡器过滤器现在支持自定义季度**

_[!UICONTROL 工作负载均衡器]_

中的过滤器 [!UICONTROL 工作负载平衡器] 现在支持自定义季度。

**更新了工作负载平衡器筛选器中“持续时间”字段的筛选器运算符**

_[!UICONTROL 工作负载均衡器]_

我们更新了过滤器运算符[!UICONTROL  工作负载平衡器] 区域 [!UICONTROL 持续时间].
