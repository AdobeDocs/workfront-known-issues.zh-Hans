## 2022年10月更新

+++**2022年10月3日的维护更新**

**当以前的作业角色发生更改时手动保存工时单**

*工时表*

如果您记录时间的作业角色已更改，并且 [!UICONTROL 手动将作业角色分配给小时条目] 设置已禁用，您必须手动保存时间条目，直到不再为已更改的作业角色记录小时为止。

## 2022年9月更新

+++**2022年9月29日的维护更新**

**关闭校样时，用户不会返回到上一页**

*验证*

在中查看校样的用户 [!DNL Workfront] 关闭校样，他们不会返回在打开校样之前所在的页面。 而是会将用户重定向到 [!DNL Workfront].

**无法在中打开校样[!DNL Workfront]**

*验证*

当用户在 [!DNL Workfront] 并尝试打开校样，校样未打开，用户将返回到 [!UICONTROL 文档详细信息] 页面。

**使用 [!UICONTROL 选项卡] key**

*工时表*

当用户填写时间表并在单元格之间导航时，使用 [!UICONTROL 选项卡] 键，则小时不会保存。 的 [!UICONTROL 自动保存] 通知不会显示在屏幕底部，如果用户刷新页面，他们可以看到未保存的小时数。

**查看包含多个页面的校样时显示空白页面**

*[!DNL Workfront Proof]*

当用户查看包含多个页面的校样时，用户可以查看页面的缩略图，但页面不会在主查看器中打开。

+++**2022年9月22日的维护更新**

**无法关闭更新流中的用户卡**

*更新*

当用户查看更新并悬停在某个名称上时，卡片中会显示有关该用户名称的详细信息，该用户名称随即打开，并且不会自动关闭。 在通过单击右上角的X手动关闭信息卡之前，页面无响应。

*更新*

*[!DNL Workfront Scenario Planner]*

+++**2022年9月8日的维护更新**

**更新了用户和角色分配字段的还原名称**

*分派*

上周临时重命名的分配字段已还原为其原始名称：

* [!UICONTROL 任务用户]
* [!UICONTROL 任务角色]

**从标题中删除项目所有者时出错**

*项目*

当用户尝试删除 [!UICONTROL 项目所有者] 在项目标题中， [!UICONTROL 项目所有者] 未删除，且用户看到以下错误消息：

`422: Invalid Parameter: ownerID value "null" /attask/api-internal/PROJ/<project ID>`

**已调整大小 [!UICONTROL 描述] 框返回原始大小**

*项目、任务和问题*

当用户调整 [!UICONTROL 描述] 框，使其变大，然后在框中开始键入，该框将返回原始大小。 用户仍可以在框中键入内容，内容会按预期进行保存

**创建任务或问题时意外退出**

*任务和问题*

当用户在项目中创建任务或问题并在创建弹出窗口之外单击时，弹出窗口将关闭且不发出警告，并且之前输入的所有信息都将丢失。

**删除了通过电子邮件将校样发送到拖放区域的功能**

*[!DNL Workfront Proof]*

自2022年9月8日星期四起，我们已删除在独立的拖放区域中通过电子邮件发送校样的功能 [!DNL Workfront Proof] 产品。

您仍可以通过其他方式使用拖放区域，即可向帐户提交新校样和新版本的校样，而无需登录到您的帐户。 请参阅 [拖放区](https://experienceleague.adobe.com/docs/workfront/using/workfront-proof/work-with-proofs-in-wf-proof/create-proofs-and-files/dropzone.html) 以了解更多信息。

+++**2022年9月6日维护更新**

**在可自定义项目标题的字段列表中添加了预计日期**

*项目*

我们已将 [!UICONTROL 预计开始日期] 和 [!UICONTROL 预计完成日期] 到使用布局模板时可自定义项目标题的字段列表。

**新限制，其中包含一条确认消息，显示添加到时间表的项目数**

*工时表*

当您选择50个以上的项目添加到时间表时，您现在会收到一条确认消息，显示要添加到时间表的项目数量，并让您有机会更改课程，而不是添加所有项目。 添加的所有项目都会自动固定到时间表上，并且需要从当前和所有将来的时间表中手动删除。

+++**2022年9月2日的维护更新**

添加 [!UICONTROL 集成] 字段中的自定义标题

*集成*

您现在可以将 [!UICONTROL 集成] 字段添加到项目的自定义标题。 添加后，该字段将显示一个链接，指向链接到中项目的外部项目 [!DNL Salesforce] 或 [!DNL Anaplan]，具体取决于您的集成。

>[!NOTE]
>
>此维护更新之前于2022年8月25日发布到预览环境，现已投入生产。

+++**2022年9月1日的维护更新**

**从委派中删除的已完成项目**

*委派*

现在，在开始委派工作项目时，只会将日期与委派日期匹配的不完整项目委派给其他用户。 如果项目在委派开始之前完成，则不会委派这些项目。 在委派时间范围内完成的项目将在委派和代理人的“主页”区域的“工作列表”中停留两周，然后自动删除它们（如果委派在这些周内未结束）。

**的元数据更新 [!DNL Adobe Workfront] 表示 [!DNL Experience Manager Assets] 和 [!DNL Assets Essentials] 集成**

*集成*

当您将资产添加到链接的文件夹时，元数据会自动推送。

以前，仅当您使用 [!UICONTROL 新增] 下拉菜单。

**无法批准或拒绝问题的小时数**

*问题*

当用户尝试批准或拒绝 [!UICONTROL 小时] 的 [!UICONTROL 批准] 和 [!UICONTROL 拒绝] 缺少按钮。

**使用模板将问题转换为项目时显示错误消息**

*问题*

在使用模板将问题转换为项目时遇到错误，此时会向用户显示一个页面，其中显示消息“[!UICONTROL 项目不再存在]“ ”，而不是解释转换失败原因的正确错误消息。

**无法为超过1.5GB的文件创建校样**

*[!DNL Workfront Proof]*

创建新校样时，如果用户上传大于1.5GB的文件，则文件名将变为红色，并且阻止创建校样。

## 2022年8月更新

+++**2022年8月25日维护更新**

**工作负载平衡器链接在功能板中显示不正确**

*仪表板*

将可共享的工作负载平衡器链接作为外部页面添加到功能板时，该链接显示不正确。 功能板不使用与链接关联的唯一视图/过滤器，而是使用最近应用的视图/过滤器来平衡工作负载。

**添加 [!UICONTROL 集成] 字段中的自定义标题**

*项目*

您现在可以将 [!UICONTROL 集成] 字段添加到项目的自定义标题。 添加后，该字段将显示一个链接，指向链接到中项目的外部项目 [!DNL Salesforce] 或 [!DNL Anaplan]，具体取决于您的集成。

>[!NOTE]
>
>此维护更新当前仅在“预览”环境中。 预览版本发布一周后，该版本将发布到生产环境。

**将问题转换为空白项目时，不会保留自定义数据**

*项目*

当用户将问题转换为空白项目（无模板）时，计算字段中的数据不会传输到新项目。

**更改项目日期时出现“时间轴规划模式”错误**

*项目*

当用户尝试更改具有 [!UICONTROL 计划模式] 设置为 [!UICONTROL 手动保存] > [!UICONTROL 时间轴规划]，则日期不会更改，用户会看到错误。

&quot;[!UICONTROL 时间轴计划模式仅在加载timelineDate时可用。 请联系 [!DNL Workfront] 这样我们就能弄明白出了什么问题，并解决它。]&quot;

**使用“月”视图打开工作负载平衡器时的一致性**

*工作负载均衡器*

现在，工作负载平衡器在 [!UICONTROL 日], [!UICONTROL 周]或 [!UICONTROL 月] 视图。 在此更新之前，已分配的项目会在 [!UICONTROL 日] 和 [!UICONTROL 周] 视图，并折叠 [!UICONTROL 月] 中。

+++**2022年8月18日维护更新**

**&quot;[!UICONTROL 添加到迭代]&quot;和&quot;[!UICONTROL 添加到看板板]“ ”选项在报表中的内联编辑任务时不可用**

*报告*

当用户查看报表中的任务列表并打开 [!UICONTROL 更多] （三点）菜单，[!UICONTROL 添加到迭代]&quot;和&quot;[!UICONTROL 添加到看板板]“ ”选项在下拉菜单中不可用。 如果在功能板中查看报表，则[!UICONTROL 添加到迭代]&quot;和&quot;[!UICONTROL 添加到看板板]“ ”选项在下拉菜单中可用。

**滚动时矩阵报表显示不正确**

*报告*

当用户查看矩阵报表并滚动时，报表的某些可视化元素可能会重叠或重复。

**[!UICONTROL 里程碑] 从工时单项目列表中删除的视图**

*工时表*

的 [!UICONTROL 里程碑] 添加项目时，已从时间表项目列表中删除视图。

**交互式校样中的超链接不处于活动状态**

*[!DNL Workfront Proof]*

当用户查看交互式校样并单击包含链接的链接或按钮时，用户不会进入该链接或按钮链接到的页面。

**新校样页面缺少文本字段**

*[!DNL Workfront Proof]*

在 [!DNL New Proof] 页面中，不显示许多文本字段（包括字段标签、下拉选项和复选框名称）。

**用户在校样中标记后不会收到通知**

*[!DNL Workfront Proof]*

当用户在校样评论中进行标记时，他们不会收到有关该评论的电子邮件通知。

+++**2022年8月12日的维护更新**

**在标题的开头处添加了新的可自定义的标题字段**

*标题*

现在，当您向可自定义的标题中添加新字段时，该字段将作为标题左侧的第一个字段添加，或紧接在 [!UICONTROL 搜索] 框。 进行此更改之前，该字段已添加为标题中的最后一个字段。

+++**2022年8月11日的维护更新**

**由于描述性文本字段的字符限制不正确，无法编辑自定义表单**

*我的组中的自定义表单*

当用户尝试编辑自定义表单，且该自定义表单具有 [!UICONTROL 描述性文本] 字段，则用户无法保存对自定义表单所做的编辑，并会看到以下错误：

&quot;以下字段无效：（字段）太长，最大512英寸

这会影响 [!UICONTROL 描述性文本] 尽管字段长度超过512个字符，但以前运行良好的字段。

**将问题转换为项目时，不会保留按部分时间隐藏的字段中的数据**

*我的组中的自定义表单*

当用户将问题转换为项目，并且问题包括一个自定义表单，该表单中的某个节点有数据（可以使用显示逻辑隐藏），则该节中的数据不会传递到新项目。

**将请求转换为项目时，不会保留按节中断隐藏的字段中的数据**

*我的组中的自定义表单*

当用户将请求转换为项目，并且请求包含一个自定义表单，该表单的节点中包含数据（可以使用显示逻辑隐藏），则该节中的数据不会传递到新项目。

**由于“描述性文本”字段，无法编辑自定义表单**

*我的组中的自定义表单*

当用户尝试编辑包含描述性文本字段的自定义表单时，该字段的标签不会填充。 用户看到错误“[!UICONTROL 此字段为必填字段]“ ”下，且用户由于出现此错误而无法编辑自定义表单。

**无法从自定义表单生成器的自定义字段中删除说明**

*我的组中的自定义表单*

当用户编辑自定义字段并尝试删除 [!UICONTROL 说明] 区域，则保存字段时不会删除文本。 用户可以编辑文本，但无法完全删除文本。

**创建请求时，团队分配不会在新请求中显示**

*请求*

当用户创建请求并为请求分配团队，然后提交请求时，不会将团队分配给所创建的请求。 这仅影响团队分配。 用户分配可按预期工作。

## 2022年7月更新

+++**2022年7月28日的维护更新**

这些问题仅在新 [!DNL Workfront] 体验。

全部 [!DNL Workfront Classic] 功能已于2022年7月14日被删除。

**从 [!UICONTROL 家庭工作列表]**

*[!UICONTROL 主页]*

当用户尝试在 [!UICONTROL 家庭工作列表]，则该项目未打开，用户会看到以下消息：

&quot;[!UICONTROL 出现错误，我们正在努力解决此问题。要继续工作，请尝试刷新此浏览器页面。]&quot;

**委派给用户的任务和问题不会显示在用户的“主页工作列表”中**

*[!UICONTROL 主页]*

用户查看其 [!UICONTROL 家庭工作列表]，则委派给用户的任何任务或问题都不会显示在列表中，并且用户可能不了解委派。

**计划报表不会发送给所有收件人**

*报告*

发送计划报表时，不会将其发送给[!UICONTROL 发送到]“ ”部分。 省略的用户是随机的，每次发送报表时可能会有所不同。

**[!UICONTROL 附加模板时无法取消选择任务]**

*模板*

当用户附加和自定义模板时，系统会要求用户取消选择他们不希望包含的任务。 但是，没有任何任务显示为已选定，用户无法取消选择它们。

**“区域设置”字段现在具有更具体的标签**

*术语*

使“[!UICONTROL 区域设置]“字段更清晰，我们更新了其标签。

* “[!UICONTROL 区域设置]“ ”字段现在标记为“[!UICONTROL 电子邮件区域设置]&quot;
* “[!UICONTROL 区域设置]“ ”字段 [!UICONTROL 设置] >[!UICONTROL 系统] >[!UICONTROL 客户信息] 区域现在标记为“[!UICONTROL 默认电子邮件区域设置]&quot;

这些字段的功能未发生更改。

**创建工时单时出现的问题**

*工时表*

已报告关于制定工时单的下列问题：

* 当用户尝试为角色创建时间表时，不会创建时间表，用户会看到错误“[!UICONTROL 未找到主键值为“XXXXXXXXXXX”的用户。]&quot;
* 当用户尝试为团队创建时间表时， [!UICONTROL typeaid] 字段不会填充团队和 [!UICONTROL 创建工时单] 按钮。


**区域 [!DNL Workfront Proof] 创建、移动或存档校样时，不进行更新**

*[!DNL Workfront]校对*

校样当前遇到索引延迟问题。 这可能会在以下方面影响用户体验：

* 功能板未显示正确的校样数
* 创建或移动校样时，文件夹不会更新
* 存档的校样将保留在活动校样列表中。

+++**2022年7月26日维护更新（热修复程序）**

这些问题仅在新 [!DNL Workfront] 体验。

全部 [!DNL Workfront Classic] 功能已于2022年7月14日被删除。

**工时单上显示的小时数与工时单列表不同**

*工时表*

当用户打开时间表以查看时，显示的小时数与用户在时间表列表中查看同一时间表时显示的小时数不同。


**转换为具有模板的项目的请求将显示请求队列中的组，而不是模板中的组**

*请求*

当用户使用模板将请求转换为项目时，新创建的项目与拥有请求队列的组相关联，而与在模板上分配的组无关联。 即使在创建项目时，与模板关联的组也会填充在 [!UICONTROL 组] 字段。
