# 通知：将通知标记为“已查看”不持久

当用户将其通知标记为已查看，然后导航到中的其他页面时 [!DNL Workfront]，通知图标仍会显示用户将其标记为可见之前存在的未读通知数，并且当用户单击该图标时，通知仍会列出。 如果用户将它们标记为已查看，并导航到其他页面或返回原始页面，则会继续此操作。

**变通方法**

刷新浏览器可允许通知按预期运行。

_首次报告于2023年11月3日。_
