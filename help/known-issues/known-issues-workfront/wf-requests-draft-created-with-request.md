# 请求：与请求一起创建的草稿导致问题

>[!NOTE]
>
>此问题已于2026年2月10日修复。

当用户在新请求Experience中创建并提交新请求时，也会创建草稿。 草稿仅显示已提交请求名称的第一个字母，尝试打开该请求会导致“无法检索”错误。

此外，如果随后删除了提交的请求，则该请求似乎会被删除，但仍会显示在Workfront的请求区域中。 如果用户尝试重新打开请求，他们会看到一条消息，指示请求不存在。

_首次报告于2025年1月6日。_
