# 请求：创建请求时，团队分配不会在新请求中显示

>[!NOTE]
>
> 此问题已于2022年8月11日修复。

当用户创建请求并为请求分配团队，然后提交请求时，不会将团队分配给所创建的请求。 这仅影响团队分配。 用户分配可按预期工作。

**解决方法**

创建请求，然后进入请求并分配团队。

_第一次报告是在2022年7月29日。_
