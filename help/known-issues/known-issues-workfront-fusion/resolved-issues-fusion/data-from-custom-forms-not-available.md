# [!DNL Workfront Fusion]:自定义表单中的数据在 [!DNL Workfront Fusion] [!DNL Workfront] 模块

>[!NOTE]
>
>此问题已于2022年6月9日修复。

当用户配置 [!DNL Workfront] 模块 [!DNL Workfront Fusion]，并尝试为模块选择输出，则自定义表单中的字段将不可见。 为以下类型创建自定义表单时会发生这种情况 [!DNL Workfront] 对象，然后向其中添加了其他类型。 [!DNL Workfront Fusion] 仅显示最初为选定对象类型创建的自定义表单中的字段。

第一次报告是在2022年4月24日。
